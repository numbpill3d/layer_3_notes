# Layer 3 Notepad - Architecture & Functionality

## Overview

Layer 3 Notepad is a thematically immersive note-taking application designed with a cyberpunk/digital ghost aesthetic. The application provides comprehensive text editing capabilities combined with organizational features like folders, tags, and backlinks, while maintaining an unsettling atmosphere through "haunting" UI elements that suggest sentience or hidden depths to the application itself.

The name "Layer 3" suggests hidden protocols and deeper realities beneath the surface level of digital interactions, creating an experience that is both functionally robust and thematically engaging.

## Core Architecture

### File Structure
- **index.html**: Core UI structure and element definitions
- **style.css**: Complete styling including theme, animations, and responsive design
- **script.js**: Application logic and functionality
- **main.js**: Electron wrapper for desktop application (when built as executable)

### Design Principles
1. **Function with Atmosphere**: All features serve practical purposes while reinforcing the haunting theme
2. **Progressive Intensity**: Haunting elements increase in frequency and intensity over time
3. **Data Ownership**: All user data stored locally (localStorage in web version, filesystem in desktop version)
4. **Responsive Design**: Adapts to different screen sizes while maintaining functionality
5. **Rich Text Capabilities**: Custom Domain-Specific Language (DSL) for formatting

## Data Management

### Storage Architecture
- **Notes**: Stored as JSON objects in localStorage (web) or files (desktop)
- **Folders**: Hierarchical structure with parent/child relationships
- **Tags**: Tracked with usage counts and note associations
- **Backlinks**: Graph of connections between notes
- **Metadata**: Custom fields for enhanced note information

### Data Structure for Notes
```json
{
  "filename": {
    "content": "Note content with formatting",
    "encrypted": false,
    "folder": "folder_id",
    "tags": ["tag1", "tag2"],
    "metadata": {
      "created": "ISO date",
      "modified": "ISO date", 
      "type": "note",
      "font": "fontname",
      "custom_field": "custom_value"
    },
    "timestamp": 1621459254000
  }
}
```

### Data Structure for Folders
```json
{
  "folder_id": {
    "name": "Folder Name",
    "parent": "parent_folder_id",
    "children": ["child_folder_id1", "child_folder_id2"]
  }
}
```

### Backlinks System
- Automatically tracks internal links between notes
- Builds a graph of note relationships
- Updates references when notes are renamed

### Formatting DSL
The application uses a custom Domain-Specific Language for text formatting:

| Syntax | Description | Example |
|---|---|---|
| `-::color::-` | Colored text | `-::red::- Text in red -::-` |
| `[glow]` | Glowing text effect | `[glow] Glowing text [/glow]` |
| `*-*-animated-*-*` | Animated glitch effect | `*-*-animated-*-* Animated text *-*-/animated-*-*` |
| `(redacted)` | Hidden text, revealed on hover | `(redacted) Hidden text (/redacted)` |
| `__underline__` | Underlined text | `__underline__ Underlined text __/underline__` |
| `<size:value>` | Text size variation | `<size:large> Larger text </size>` |

### Linking System
- Internal links between notes using `[[Note Name]]` syntax
